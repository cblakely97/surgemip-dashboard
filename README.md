# STOFS2D-Global for SurgeMIP -- Dashboard

Static website showing progress and validation results for the STOFS2D-Global storm surge simulations contributing to [SurgeMIP](https://surgemip.org).

## Pages

- **About** (`index.html`) -- project overview, model configuration, scenario descriptions, forcing pipeline, and validation methodology
- **Progress** (`progress.html`) -- per-year completion chart for each scenario (CFS reanalysis, HadGEM present/future)
- **Station Comparison** (`comparison.html`) -- interactive Leaflet map of GESLA tide gauge stations with ADCIRC vs observation time series and residuals

## Local preview

The site uses `fetch()` to load JSON data, which requires HTTP (not `file://`). Run a local server:

```bash
cd surgemip-dashboard
python3 -m http.server 8000
```

Then open http://localhost:8000. The progress chart renders fallback (empty) bars when opened via `file://`.

## Data

All data lives in `data/` and is generated by scripts in the [STOFS2D-Global_SurgeMIP](https://github.com/cblakely97/STOFS2D-Global_SurgeMIP) repo:

```bash
# Update scenario progress bars
python postprocessing/update_progress.py

# Extract station time series and GESLA comparisons
python postprocessing/extract_station_data.py \
  --years 1979 \
  --gesla-dir /path/to/GESLAv4/ \
  --adcirc-dir /path/to/output/ \
  --output-dir /path/to/surgemip-dashboard/data/
```

Outputs:
- `data/progress.json` -- scenario completion status
- `data/station_node_map.json` -- GESLA station to mesh node mapping (631 stations)
- `data/stations.json` -- station metadata and validation metrics
- `data/timeseries/<station_id>.json` -- per-station ADCIRC + GESLA time series

## Stack

Plain HTML/CSS/JS, no build step. [Leaflet](https://leafletjs.com/) for maps, [Plotly.js](https://plotly.com/javascript/) for charts.
